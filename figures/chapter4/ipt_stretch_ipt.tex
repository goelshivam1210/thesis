    \caption{Conceptual overview of the integrated planning task (IPT) formalism. Each symbolic operator $\Operator_i$ is mapped to a subsymbolic executor $\Executor_i = \langle I_{\Executor_i}, \Policy_{\Executor_i}, \beta_{\Executor_i} \rangle$ via the executor function~$e$. The abstraction function $\Abstraction : \States \rightarrow \SymStates$ maps environment states back to symbolic states for the planner. \textbf{(a)}~In a working IPT, all executors realize their associated operators and the agent reaches the goal~$\Goal$. \textbf{(b)}~When novelty causes an operator to fail, the IPT becomes a Stretch-IPT: the task remains solvable but the current executor mapping is broken. RAPid-Learn solves the executor discovery problem by constructing a new executor~$\Executor'_2$ through reinforcement learning, restoring solvability.}
